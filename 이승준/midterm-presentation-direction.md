## 1. Pump.fun 플랫폼 개요

### 1-1. 플랫폼 소개

Pump.fun은 솔라나(Solana) 블록체인 위에서 작동하는 밈코인 원클릭 런칭 플랫폼이다.

- 누구나 코딩 없이, 토큰 이름과 이미지만 입력하면 수 초 만에 토큰 발행 가능
- 하루 평균 **2만 개 이상** 토큰 생성
- 전통 금융시장의 IPO(수년) vs Pump.fun(수 초)

### 1-2. 토큰 라이프사이클

```
토큰 생성 → 본딩 커브 거래 → 졸업 (Graduation) → DEX 거래
 (수 초)     (수 분~수 시간)     (자동)              (무기한)
```

| 단계 | 설명 |
|------|------|
| **토큰 생성** | Creator가 이름·이미지 입력하여 발행 |
| **본딩 커브 거래** | 스마트 컨트랙트가 자동 유동성 공급. 매수→가격↑, 매도→가격↓ |
| **졸업 (Graduation)** | 시가총액 임계치 도달 시 Raydium(솔라나 DEX) 자동 상장 |
| **DEX 거래** | 일반 탈중앙거래소에서 자유 거래 |

모든 토큰이 동일한 라이프사이클을 거친다. 이 고정된 구조가 체계적 분석을 가능하게 한다.

### 1-3. 규모감: 실제 숫자

은성준 팀원이 2025년 1월 5일 하루 동안 Pump.fun에서 출시된 토큰을 분석한 예비 결과:

| 지표 | 수치 |
|------|------|
| 해당일 생성 토큰 수 | 47,247개 |
| RUG로 라벨링된 토큰 | 163개 |
| HEALTHY로 라벨링된 토큰 | 45개 |
| 라벨링 기준 미달 (분석 대상 외) | 47,039개 |

- **RUG 기준**: 10분 내 시가총액 정점 도달(max_mcap_10m > 80% of max_mcap) + 1시간 내 폭락(mcap_1h < 20% of max_mcap)
- **HEALTHY 기준**: 1시간 시점 성장 유지 + 4시간 시점 가치 유지
- 라벨링 기준 미달 토큰의 대다수는 거래량이 극히 적어 분석 의미가 없는 토큰으로 추정

> 하루 4.7만 개 토큰 중, 의미 있는 거래가 발생하는 토큰조차 대부분 러그풀로 귀결된다.

## 2. 문제의식: 구조적 불공정거래 취약성

### 2-1. 왜 Pump.fun이 불공정거래에 취약한가

| 구조적 특성 | 불공정거래 취약점 |
|------------|----------------|
| Creator = 최대 보유자 | 극단적 정보 비대칭 — 매도 의도를 Creator만 알고 있음 |
| 본딩 커브의 얕은 유동성 | 소액으로도 가격 조종 가능 |
| 솔라나 블록 타임 ~0.4초 | 봇이 일반 투자자보다 수십 배 빠르게 선점 매수 |
| 지갑 생성 무료·무제한 | 한 사람이 수백 개 지갑으로 위장 거래 가능 |
| KYC 없는 익명 환경 | 행위자 식별 및 사후 추적이 극도로 어려움 |

### 2-2. 감시 공백

전통 금융시장에는 거래소 감시부서와 금융감독원이 이상거래를 탐지한다. Pump.fun에는 그런 감시 체계가 전혀 없다. 우리 프로젝트는 이 공백에서 출발한다.

### 2-3. 왜 블록체인이어야 하는가

| | 전통 금융시장 | Pump.fun |
|--|------------|----------|
| 문제의 원인 | 내부자 정보 남용 | 무허가 토큰 발행 + 익명성 |
| 감시 데이터 | 거래소만 접근 가능 | **공개 원장 — 누구나 조회 가능** |
| 감시 주체 | 금감원, 거래소 | 없음 → **우리가 시도** |

블록체인이 불공정거래를 가능하게 하는 동시에, 탐지할 수 있는 데이터도 제공한다. Dune Analytics로 솔라나 온체인 데이터를 SQL 쿼리하면, 토큰의 전체 라이프사이클을 재구성할 수 있다.

## 3. 불공정거래 유형 분류

### 3-1. 법학적 기준

Pump.fun의 불공정거래를 전통 금융시장의 불공정거래 법리에 대응시켜 분류한다.

| 법적 유형 | 한국 자본시장법 | 한국 가상자산이용자보호법 | 미국 |
|----------|-------------|-------------------|------|
| 시세조종 | §176 | 제11조 | CEA §6(c)(1), Wire Fraud |
| 부정거래 | §178 | 제12조 | Wire Fraud (18 U.S.C. §1343) |
| 미공개정보이용 | §174 | 제10조 | — (Pump.fun에서는 전통적 의미의 insider trading 성립 어려움) |

### 3-2. Pump.fun에서 발생 가능한 6가지 유형

Pump.fun의 고정된 라이프사이클(생성→본딩커브→졸업→DEX) 안에서, 발생 가능한 불공정거래 패턴을 아래 6가지로 분류했다.

| # | 유형 | 설명 | 라이프사이클 단계 | 법적 분류 |
|---|------|------|----------------|---------|
| A | **Pump-and-Dump** (러그풀) | Creator 또는 초기 보유자가 가격을 끌어올린 뒤 대량 매도 | 본딩 커브 → 졸업 직후 | 시세조종 |
| B | **시리얼 런칭** | 동일 지갑이 토큰을 반복 생성, 대부분 실패, 일부에서만 차익 | 생성 | 부정거래 |
| C | **봇 선점매수** (Sniping) | 토큰 생성과 동일 블록에서 봇이 자동 매수, 가격 우위 선점 | 생성 직후 | 규제 공백 (HFT 유사) |
| D | **동시 다중지갑 매도** | 한 주체가 수십~수백 지갑에 분산 보유 후 같은 슬롯에서 동시 매도 | 본딩 커브 / 졸업 후 | 시세조종 |
| E | **자전거래** (Wash Trading) | 동일 주체가 여러 지갑으로 사고팔아 거래량 부풀림 | 전 단계 | 시세조종 |
| F | **지갑 분산** (Wallet Splitting) | 대량 매도를 숨기기 위해 토큰을 여러 지갑으로 분산 후 개별 매도 | 전 단계 | 부정거래 |

각 유형의 온체인 탐지 가능성 판정은 최종 발표에서 상세히 다룰 예정이다.

## 4. 법적 근거: 왜 이 분석이 법적으로 의미 있는가

### 4-1. 한국 — 가상자산이용자보호법 (2024. 7. 시행)

자본시장법의 불공정거래 3대 유형을 가상자산에 직접 이식한 법률이다. **시장 규모에 따른 적용 제한이 없어**, Pump.fun 밈코인에도 원칙적으로 적용 가능하다.

| 조항 | 내용 | 대응하는 Pump.fun 유형 |
|------|------|---------------------|
| 제10조 | 미공개중요정보 이용 금지 | — (전통적 insider trading 구조 부재) |
| 제11조 | 시세조종행위 금지 | A(러그풀), D(동시매도), E(워시트레이딩) |
| 제12조 | 부정거래행위 금지 | B(시리얼 런칭), F(지갑 분산) |

다만 실무적으로는 역외 적용(제23조)과 해외 DeFi 프로토콜에 대한 집행 가능성이 별도 쟁점이다.

### 4-2. 미국 — 세 가지 규율 경로

미국은 가상자산 불공정거래를 세 가지 독립적인 경로로 규율한다.

```
가상자산 불공정거래
    │
    ├─ 증권인가? ── YES → SEC (Rule 10b-5 등)
    │             NO ↓
    ├─ 상품인가? ── YES → CFTC (CEA §6(c)(1))
    │             불확실 ↓
    └─ 기망인가? ── YES → DOJ (Wire Fraud, 18 U.S.C. §1343)
                          ↑ 자산 분류와 무관
```

**SEC 경로**: 토큰이 증권(security)이어야 적용. Pump.fun 밈코인은 "발행자의 경영적 노력에 대한 수익 기대"가 없으므로 Howey 테스트 충족 가능성 낮음. 2026년 3월 SEC/CFTC 공동 Token Taxonomy(Release Nos. 33-11412)도 밈코인을 Digital Securities로 분류하지 않음. **→ 현실적 적용 가능성 낮음**

**CFTC 경로**: 토큰이 상품(commodity)이어야 적용. 밈코인이 commodity에 해당하는지는 자동 결정되지 않으며, Token Taxonomy에서 명시 지정된 16개 자산에도 포함되지 않음. **→ 이론적으로 가능하나 현실적으로 불확실**

**DOJ Wire Fraud 경로**: 자산 분류와 무관하게, "통신수단을 이용한 기망에 의한 금전 취득"이면 18 U.S.C. §1343 적용 가능. **→ Pump.fun 같은 해외 DeFi 밈코인 시장에서 현실적으로 가장 유력한 규율 수단**

### 4-3. 프로젝트에 대한 법적 시사점

1. **"증권이 아니면 합법"이 아님**: 한국법은 자산 분류 무관하게 가상자산 전체에 적용. 미국도 wire fraud 경로가 존재.
2. **온체인 증거의 법적 가치**: wire fraud 기소에는 "기망의 계획(scheme to defraud)"을 입증하는 증거가 필요. 대시보드가 추출하는 행동 패턴(동시 매도, 시리얼 런칭 등)은 이러한 기망을 뒷받침하는 정황 증거로 기능할 수 있다.
3. **한미 비교의 학술적 가치**: 한국(분류 무관 적용) vs 미국(세 경로 분기)의 구조적 차이 자체가 분석 포인트.

## 5. 접근 방법: 법학적 분류 × 온체인 데이터 탐지

### 5-1. 기존 접근의 한계

| 접근 방식 | 한계 |
|----------|------|
| 데이터만 보는 접근 | 가격 이상치를 탐지하지만, 법적으로 무엇인지 모름 |
| 법만 논하는 접근 | 규제를 논하지만, 실제 데이터로 검증하지 않음 |

### 5-2. 우리의 접근: 3단계 프레임워크

```
① 유형 분류
   불공정거래 법리를 기준으로 Pump.fun 라이프사이클별 패턴을 6가지 유형으로 분류
        ↓
② 탐지 가능성 판정
   각 유형이 Dune의 공개 테이블만으로 탐지 가능한지 단계별 판정
        ↓
③ 교차 검증
   RUG/HEALTHY 라벨링 데이터를 정답지로 활용, 탐지 지표의 유효성을 통계 검증
```

## 6. 대시보드의 이중 역할

### 6-1. 사전 시그널링 (Yellow Flag System)

실시간 온체인 데이터에서 의심스러운 토큰에 경고 신호를 부여한다.

관찰 가능한 신호 후보:
- Creator의 과거 토큰 생성 이력이 비정상적으로 많은가?
- 토큰 생성 직후 극초기 매수가 집중되는가? (봇 추정)
- 짧은 시간 안에 다수 지갑이 동시 매도하는가?

개별 신호는 확정적 증거가 아니지만, 복합적으로 나타나면 실질적 경고가 될 수 있다.

### 6-2. 사후 증거 기준 (Forensic Evidence Framework)

러그풀 발생 후 "이것이 불공정거래였다"를 입증할 때 참조할 수 있는 데이터 기반 증거 체계를 제시한다.

- 가격 궤적 재구성 (생성 → 정점 → 폭락)
- Creator 지갑의 행동 패턴 기록
- 동시 매도 등 정황 증거의 통계적 비우연성 제시

가상자산이용자보호법(한국), wire fraud(미국) 모두 온체인 증거 체계의 필요성을 현실적으로 뒷받침한다.

## 7. 팀 구조와 개별 분석 상세

### 7-1. 역할 분담

| 팀원 | 분석 초점 | 핵심 산출물 |
|------|---------|----------|
| 이승준 | 프로젝트 기획 + 발표 방향 설계 + 법률 리서치 | 중간발표 방향 보고서, 법적 프레임워크 정본, slidev 슬라이드 |
| 은성준 | RUG/HEALTHY 사후 라벨링 + 특징 분석 | 라벨링 데이터셋, 연구 보고서 |
| 조석희 | 졸업 전 행동 패턴 시각화 | Dune 대시보드 (pre-graduation behavior) |
| 이민재 | 데이터 인프라 구축 + 소스 검증 | Dune CLI 환경, 테이블 매핑, query spec |
| 박원현 | 동시 매도 탐지 대시보드 + 통합 프레임워크 초안 | 프레임워크 문서, Dune 대시보드 (슬롯 기반 탐지) |

### 7-2. 이승준 (lees2345) — 프로젝트 기획 + 법률 리서치 + 발표

**목적**: 프로젝트 전체 방향 설계, 불공정거래 관련 법률 리서치, 중간/최종 발표 제작.

**주요 작업**:

*프로젝트 기획*
- 각 팀원의 개별 분석을 하나의 프레임워크로 통합하는 방향성 설계
- "법학적 분류 × 온체인 데이터 탐지"라는 프로젝트 핵심 컨셉 수립
- 대시보드의 이중 역할(사전 시그널링 / 사후 증거 기준) 개념 정립
- 교차 검증 설계: 은성준 라벨 데이터를 정답지로 활용하는 검증 구조

*불공정거래 법률 리서치*
- 한국 자본시장법(§174/§176/§178)과 가상자산이용자보호법(제10~12조)의 Pump.fun 적용 분석
- 미국 규율 체계 정리: SEC/CFTC/DOJ 세 경로 프레임워크 구축
- Wire fraud(18 U.S.C. §1343)가 자산 분류와 무관하게 적용 가능하다는 핵심 논점 도출
- 한미 비교: 한국(분류 무관 적용) vs 미국(세 경로 분기)의 구조적 차이 분석

**프레임워크에서의 역할**: 개별 분석들을 연결하는 "포장지" 역할. 각 팀원의 산출물이 법학적으로 어떤 의미를 갖는지, 전체 프로젝트가 왜 중요한지를 설명하는 내러티브와 법적 근거를 제공한다.

### 7-3. 은성준 — RUG/HEALTHY 라벨링 데이터

**목적**: 사후적으로 "이 토큰은 러그풀이었다/건전했다"를 판정하는 기준을 수립하고, 이를 다른 분석의 교차 검증 정답지로 제공.

**방법**:
- 데이터 소스: Dune Analytics (`pumpdotfun_solana` 테이블)
- 대상: 2025년 1월 5일 Pump.fun 출시 토큰 47,247개
- RUG 판정: 10분 내 시가총액 정점(max_mcap_10m > 80% of max_mcap) + 1시간 내 폭락(mcap_1h < 20% of max_mcap)
- HEALTHY 판정: 1시간 시점 성장 유지 + 4시간 시점 가치 유지
- 결과: RUG 163개, HEALTHY 45개 라벨링

**프레임워크에서의 역할**: 이 라벨이 "정답지"가 된다. 다른 팀원의 지표(졸업 속도, 초기 매수 집중도, 동시 매도 패턴 등)가 RUG와 HEALTHY를 유의미하게 구분하는지를 검증하는 데 사용된다.

### 7-4. 조석희 (soarewee) — Pre-Graduation Behavior 대시보드

**목적**: 졸업 전 단계(본딩 커브)에서의 행동 패턴을 모집단 수준에서 시각화.

**Dune 대시보드 주요 차트**:
- 런칭 코호트별 졸업률
- 졸업까지 소요 시간 분포
- 초기 거래자 집중도와 졸업의 관계
- 초기 거래 속도와 졸업의 관계

**참조 테이블**: `pumpdotfun_solana.pump_call_create`, `pump_amm_evt_createpoolevent`, `dex_solana.trades`

**프레임워크에서의 역할**: 유형 A(러그풀)의 졸업 속도, 유형 C(봇 선점매수)의 초기 매수 집중도에 대한 모집단 분포를 이미 시각화해놓은 상태다. 은성준의 라벨과 결합하면 "RUG 토큰이 이 분포의 어디에 위치하는가"를 보여줄 수 있다.

### 7-5. 이민재 — 데이터 인프라 및 테이블 매핑

**목적**: 전체 분석의 데이터 기반 구축.

**주요 작업**:

- Dune CLI + Dune Skills 기반 작업 환경 확립
- 핵심 테이블 확인 및 스키마 문서화

**핵심 Dune 테이블**:

| 테이블 | 내용 | 용도 |
|--------|------|------|
| `pumpdotfun_solana.pump_call_create` | 토큰 생성 이벤트 (creator 주소, 시각) | 시리얼 런칭 분석, 토큰 생성 시점 기준 |
| `pump_amm_evt_createpoolevent` | 졸업 이벤트 (Raydium 상장) | 졸업 속도, 졸업 여부 판정 |

### 7-6. 박원현 (pwh620) — 통합 프레임워크 초안 + 동시 매도 탐지 대시보드

**목적 1 — 통합 프레임워크 초안**: 6가지 유형 분류, 탐지 가능성 판정, 교차 검증 설계의 초안을 작성. (이승준이 법률 리서치를 보강하여 최종 프레임워크로 완성)

**목적 2 — 동시 매도 탐지 대시보드**: 유형 D(동시 다중지갑 매도)를 직접 탐지하는 Dune 대시보드 구축.

**대시보드 핵심 로직**:
- `dex_solana.trades`에서 **(슬롯, 토큰) 쌍으로 그룹핑**
- 같은 0.4초(1 슬롯) 안에 다수 지갑이 매도하는 이벤트를 탐지
- 지갑 간 자금 흐름을 직접 추적하지 않고, **"동시성"이라는 행동 패턴 자체를 정황 증거로 활용**

**3대 지표**:
- **Price Drop %**: 1슬롯 내 가격 하락률
- **Sell Ratio %**: 유통 물량 중 풀로 복귀한 비율
- **SOL Sold**: 매도로 풀에서 유출된 SOL 총량 (피해 규모)

**왜 이 접근이 중요한가**: 유형 E(워시 트레이딩), F(지갑 분산)를 탐지하려면 지갑 간 자금 흐름 그래프 분석(솔라나 베이스 레이어)이 필요하며, 이는 Dune의 앱 레이어 테이블로는 불가능하다. 박원현의 접근은 이 한계를 **우회**한다. 직접 증거(지갑 연결) 대신 정황 증거(동시성)를 사용하여, 앱 레이어 데이터만으로도 다중지갑 매도를 간접 탐지한다. 이는 전통 금융에서 시세조종 입증 시 "거래 패턴의 인위성"에 의존하는 것과 동일한 논리 구조다.

### 7-7. 분석 흐름

```
[이승준: 프로젝트 기획 + 법률 리서치] ──→ 전체 방향 설계, 법적 프레임워크, 발표 제작
        ↓ 프레임워크 제공
[은성준: RUG/HEALTHY 라벨] ──→ 정답지 제공
        ↓
[조석희: 졸업 전 행동 분포] ──→ 모집단 분포에서 RUG 위치 확인
        ↓
[이민재: Dune 인프라] ──→ 쿼리 실행 기반
        ↓
[박원현: 프레임워크 초안 + 동시매도 탐지] ──→ 통합 분석 + 교차 검증
```

## 10. 예상 질문 대비

**Q: 밈코인에 불공정거래 규제가 적용되나?**
→ 한국: 가상자산이용자보호법(2024.7.) 제10~12조, 시장 규모 무관. 미국: DOJ wire fraud (자산 분류 무관), Operation Token Mirrors(2024) 선례.

**Q: 탐지가 안 되는 유형이 있으면 의미가 없지 않나?**
→ "무엇을 탐지할 수 없는가"를 명확히 하는 것 자체가 규제적 함의. 최종 발표에서 구체적으로 다룰 예정.

**Q: Yellow flag가 실제로 효과가 있는가?**
→ 교차 검증 단계에서 RUG/HEALTHY 그룹 간 통계적 차이를 검증할 계획. 최종 발표의 핵심.

**Q: 다른 플랫폼에도 적용 가능한가?**
→ 법학적 분류(유형 체계)는 DeFi 토큰 런칭 플랫폼 전반에 적용 가능. 탐지 방법론은 플랫폼별 조정 필요.

**Q: 미국에서 증권이 아닌 가상자산도 규제할 수 있나?**
→ Wire fraud는 자산 분류와 무관. 다만 DeFi 환경에서 "기망"의 범위가 아직 판례상 정립 중 (Eisenberg 파기, Chastain 파기). 한국법은 분류 무관 적용으로 이 문제 없음.
